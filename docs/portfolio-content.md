# Portfolio Website — Content Guide

A reference for what content to include in each section of a software engineer's personal portfolio.

---

## 1. Hero / Introduction

The first thing visitors see. Should be clear and compelling.

- **Full name**
- **Job title / role** (e.g. "Full-Stack Engineer", "Backend Engineer")
- **Tagline** — one sentence describing what you do or your specialty
- **Short bio** (2–3 sentences max) — who you are and what you build
- **Profile photo** — professional but approachable
- **CTA buttons** — e.g. "View my work" and "Contact me" or "Download CV"
- **Social links** — GitHub, LinkedIn, Twitter/X (optional)

---

## 2. About Me

A human connection — goes deeper than the hero tagline.

- **Background story** — where you're from, how you got into engineering
- **What drives you** — what problems you enjoy solving, what excites you
- **Work style / values** — how you approach your craft (e.g. clean code, user-first thinking)
- **Personal interests** (optional) — hobbies that show personality
- **Location / availability** — city, open to remote, open to opportunities

---

## 3. Skills / Tech Stack

Demonstrates technical breadth. Group by category for readability.

- **Languages** — e.g. TypeScript, Python, Go, Java
- **Frontend** — e.g. React, Next.js, Vue, Tailwind CSS
- **Backend** — e.g. Node.js, Express, FastAPI, PostgreSQL, Redis
- **DevOps / Infra** — e.g. Docker, AWS, CI/CD, Kubernetes
- **Tools** — e.g. Git, Figma, VSCode, Postman
- **Other** — e.g. testing frameworks, CMS, mobile (React Native, Flutter)

> Tip: Only list technologies you can confidently discuss in an interview.

---

## 4. Work Experience

A timeline of professional roles — most recent first.

For each role include:
- **Job title**
- **Company name** (with link to company website)
- **Duration** — start month/year to end (or "Present")
- **Key contributions** — 2–4 bullet points of what you built/improved/achieved
- **Technologies used** — the stack relevant to that role
- **Impact** — quantify where possible (e.g. "reduced load time by 40%")

---

## 5. Projects / Portfolio

Showcase of your best work — personal or professional.

For each project include:
- **Project name**
- **Short description** — what it is and why it was built
- **Technologies used** — language, framework, tools
- **Your role** — solo, lead, contributor
- **Links** — live demo URL and/or GitHub repo
- **Screenshot or preview** (optional but strongly recommended)

> Tip: 3–6 quality projects is better than a long list of incomplete ones.

---

## 6. Education

Academic and professional credentials.

- **Degree** — field of study, institution, graduation year
- **Certifications** — e.g. AWS Certified, Google Cloud, Meta Front-End
- **Notable courses** — if relevant (e.g. Algorithms, Distributed Systems)
- **Honors / GPA** (optional) — only include if noteworthy

---

## 7. Contact

Make it easy for people to reach you.

- **Email address** (primary contact method)
- **LinkedIn profile** URL
- **GitHub profile** URL
- **Contact form** (optional) — name, email, message fields
- **Availability note** (optional) — e.g. "Open to full-time roles" or "Available for freelance"
